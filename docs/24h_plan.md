# 24시간 타임박스 플랜

각 구간에 **한 사람의 진행 담당**을 지정. 시간 초과 시 즉시 스코프 축소.

---

## Phase 0 · 문제 정의 (0h ~ 2h)
- [ ] 팀 킥오프 (30분): 각자 스킬셋, 관심 트랙 공유
- [ ] 스폰서 트랙 확인 → 1개 타겟팅
- [ ] 문제 정의 1문단으로 씀 ("우리는 X 사용자의 Y 문제를 Z 방식으로 푼다")
- [ ] MVP 스펙 정의: 데모에 반드시 있어야 할 3가지 기능
- [ ] 데이터 소스 확정 (HF Datasets? API? 자체 수집?)
- [ ] 아키텍처 다이어그램 화이트보드 (프론트↔백↔모델)
- [ ] 팀 git 저장소 만들기 → 이 폴더에 원격 붙이기
- **산출물**: `docs/hackathon_info.md` 하단에 문제 정의 1문단 추가

## Phase 1 · MVP 뼈대 (2h ~ 8h)
- [ ] 데이터 파이프라인: raw → processed 스크립트
- [ ] 베이스라인 모델 학습 (가장 작은 모델로 빠르게)
- [ ] 프론트/백엔드 최소 골격 (다른 팀원이 병렬)
- [ ] 첫 end-to-end 데모 (모델 output이 프론트에 뜬다)
- **체크포인트**: 8h 시점에 "동작하는 것" 이 있어야 함. 없으면 스코프 반토막.

## Phase 2 · 실험·튜닝 (8h ~ 16h)
- [ ] 병렬 실험 launch (branch A/B/C 동시)
- [ ] eval harness 로 정량 비교
- [ ] 데이터 aug / 프롬프트 엔지니어링 / LoRA 튜닝 중 가장 임팩트 큰 것 하나 골라 집중
- [ ] WandB 리더보드로 실험 랭킹 유지
- **체크포인트**: 16h 시점에 최종 모델 후보 확정

## Phase 3 · 통합·데모 (16h ~ 22h)
- [ ] 최종 모델 프로덕션 서빙 (FastAPI/gradio/streamlit 중 팀 결정)
- [ ] 프론트 폴리싱, UX 갈아엎기 (5분 안에 이해되는 화면)
- [ ] 데모 시나리오 스크립트 2개 준비 (성공/실패 폴백)
- [ ] 슬라이드 초안: 문제·솔루션·아키텍처·데모·다음 스텝
- **체크포인트**: 22h 시점에 데모 영상 녹화 완료 (라이브 사고 대비 백업)

## Phase 4 · 피치 리허설 (22h ~ 24h)
- [ ] 전체 팀 리허설 2회 (각 3-5분)
- [ ] Q&A 예상 질문 10개 답변 준비
- [ ] 슬라이드 최종 저장 (PDF + 원본)
- [ ] 제출 폼 마감 확인
- **산출물**: 제출 완료 스크린샷

---

## 위기 대응 카드
- **모델이 안 학습됨** → 프리트레인 체크포인트로 fallback, zero-shot 데모로 전환
- **데이터가 부족함** → HF Datasets 검색 + synthetic data 생성 (GPT-4로 라벨링)
- **GPU가 뻗음** → Colab / RunPod 즉시 임대 (portability_guide.md 시나리오 1 참조)
- **팀원 다운** → 태스크 재분배, 스코프 축소 (MVP 3가지 → 2가지)
